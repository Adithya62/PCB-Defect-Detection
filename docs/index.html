Project Overview

This project focuses on semantic-aware PCB defect detection using deep learning, with a planned VGG16 + Fusion classifier that combines deep and shallow features. The goal is to improve tiny defect detection on printed circuit boards through balanced feature fusion and robust classification.

Week 5: VGG16 + Fusion

DONE ▼
  • Built a VGG16 + shallow-feature fusion model (512 deep + 17 shallow = 529 features)
  • Initial fusion scored 99.41% accuracy but showed strong defect bias in predictions
  • Observed bias: 85.6% predicted as defect vs true defect rate 49.9% (many normal boards flagged)
  • Applied PCA to deep features (32 components, ~95% variance) and fused to 49 features total
  • Balanced fusion results: 97.56% accuracy, 100% precision, 95.11% recall, zero false positives
  • Bias corrected: defect prediction rate dropped to 47.5% (close to true 49.9%)
  • Selected 32 PCA components as the best trade-off between bias reduction and recall
  • Recommended post-PCA threshold: 0.40 (recall ~97%, precision ~98%)

🔑 Key Findings

  1. VGG16 patch classifier achieved 98% accuracy but required pre-cropped patches
  2. Binary classification at patch-level fundamentally limited (37% false negatives)
  3. VGG + shallow fusion reached 99.41% accuracy but showed strong defect bias (85.6% predicted defective)
  4. PCA-balanced fusion (32+17 features) removed bias and achieved 100% precision with zero false positives
  5. Optimal PCA components: 32 (best balance of variance, bias reduction, and recall)
  6. Prediction distribution checks exposed hidden bias that accuracy alone masked
  7. Threshold 0.40 after PCA improved recall to ~97% while keeping precision ~98%
  8. Shallow features gained meaningful influence (35%) after dimensionality reduction

📝 Lessons Learned

  • Always start with simple baseline before complex models
  • Full images provide better context than isolated patches
  • High accuracy can hide severe bias, so always check prediction distribution
  • Feature dominance matters; deep features can overwhelm shallow ones
  • PCA balancing is effective for reducing bias while preserving performance
  • Zero false positives can be achieved by balancing feature influence
  • Document everything weekly - saves time during final reporting
